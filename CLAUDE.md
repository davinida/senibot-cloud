# SeniBot Cloud — 작업 규칙

이 저장소에서 작업할 때 반드시 지켜야 하는 운영 규칙과 컨벤션이다.

## 운영 규칙

- **RDS 비용 관리**: 매 작업 세션 시작 시 RDS 인스턴스를 `start` 하고, 작업 종료 시 `stop` 한다. (비용 절감 목적이며, 시작을 잊지 말 것.)
- **리전 고정**: 모든 AWS 리소스는 `ap-northeast-2`(서울) 리전에 생성한다.
- **민감정보 비커밋**: AWS 자격증명, OAuth 토큰, DB 비밀번호 등 민감정보는 절대 git에 커밋하지 않는다. `.env`와 AWS Secrets Manager로 관리한다.

## 기준 문서

- 구현 작업의 기준 문서는 `docs/implementation-plan.md` 이다. 작업을 시작하기 전 반드시 이 문서를 참조한다.
- 설계 변경이나 의사결정이 있을 경우 해당 문서를 함께 갱신한다.
